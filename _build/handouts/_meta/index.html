So, if you look at this week’s assignment, it doesn’t require you to do any actual coding. It focuses on commenting your code. Ever heard the phrase ‘go slow to go fast’? This is a great example of that. Front load your work so that you:

  • Are kind to your future self. Comments help you remember what each part of your code does so that changing it is easier,

  • Get a more in-depth understanding of the pieces of a program instead of what a program does as a whole. This’ll help you break down a problem and learn to employ only the parts you need.

  • Get into the good habit of comments! I’ve seen some of you already starting to comment thoroughly–that’s great! Comments help you while you’re coding, as well as later.

Also, when we wrote this exercise, it focused on lists. Now that you’ve started working with dictionaries, it’s a great way to practice implementing dictionaries and tying what you’ve learned about dictionaries to our previous studies on lists.
